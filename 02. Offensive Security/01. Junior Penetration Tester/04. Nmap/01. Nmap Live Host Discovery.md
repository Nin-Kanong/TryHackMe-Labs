## Task 1: Introduction



### What is Nmap?
**Nmap (Network Mapper)** is a free, open-source network scanning tool created by Gordon Lyon (Fyodor).

It is used by:
- Network administrators
- Penetration testers
- Security analysts
- Ethical hackers

Its main purposes are to:
- Discover live hosts
- Scan ports
- Detect operating systems
- Identify running services and versions
- Perform network reconnaissance
- Run NSE (Nmap Scripting Engine) scripts


### Nmap Scan Process

Nmap usually follows these nine steps.

<img width="1200" height="552" alt="image" src="https://github.com/user-attachments/assets/a5524f2f-0f8f-4aa2-90e9-7f535433b507" />



### Nmap Flow Cycle for Host Discovery

- Host Discovery
   - First step: identify which systems are alive. Scanning offline hosts wastes time and creates unnecessary noise.

- Link Layer Broadcasts  
   - On the same subnet, Nmap can send ARP requests. Any host that replies is confirmed alive.

- ICMP Echo & Variants  
   - Across different segments, Nmap uses ICMP echo requests (like ping), timestamp requests, or address mask requests to detect live hosts.

- TCP SYN/ACK Ping  
   - At the transport layer, Nmap sends SYN or ACK packets to common ports. If the host responds, it’s alive.
         - Privileged scans (run as root/admin) can craft raw packets.
         - Unprivileged scans rely on system calls like connect(), which are less flexible.

- ICMP Port Unreachable  
  - If Nmap probes a closed port, the host may reply with “ICMP port unreachable,” confirming it’s online.


### Flow Cycle Overview

| **Step**         | **Technique**   | **Purpose**                     |
| ---------------- | --------------- | ------------------------------- |
| ARP Scan         | Link layer      | Detect hosts in local subnet    |
| ICMP Echo        | Network layer   | Detect hosts across segments    |
| TCP SYN/ACK      | Transport layer | Detect hosts via port responses |
| ICMP Unreachable | Error handling  | Confirm host presence           |

### What is a Network Segment?
A **network segment** is a group of devices connected to the same physical network.

Example:
<img width="1536" height="1024" alt="image" src="https://github.com/user-attachments/assets/075cf1da-cd1f-4b85-86ba-44d3b5b24a58" />


Devices in the same segment communicate directly.



### What is a Subnet?
A **subnet (subnetwork)** is a logical division of an IP network.

Each subnet has its own IP range.

Example:
````
10.1.100.0/24
````
Meaning:
- Network Address: ``10.1.100.0``
- Usable Hosts: ``10.1.100.1 – 10.1.100.254``
- Broadcast: ``10.1.100.255``



### The Four Networks in the Diagram
| Network   | Subnet            |
| --------- | ----------------- |
| Network A | **10.1.100.0/24** |
| Network B | **10.2.0.0/16**   |
| Network C | **10.3.200.0/24** |
| Network D | **10.4.0.0/16**   |

Each network is separate.


### Router
The **router** connects different networks together.
<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/7c724a5a-9686-4345-86b6-f569e6e027be" />

Without a router, devices in different subnets cannot communicate.




### Firewall
A firewall filters network traffic between subnets.
<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/c82bd96d-942e-436a-8615-e65978e93f1c" />


It can:
- Allow traffic
- Block traffic
- Restrict access between networks



### Why is This Important for Nmap?
Nmap uses different host discovery techniques depending on whether the target is on the **same subnet** or a **different subnet**.

#### Same Subnet
Nmap uses:
````
ARP Scan
````
Fast and reliable.


#### Different Subnet
Nmap cannot use ARP.

Instead it uses:
- ICMP Ping
- TCP Ping
- UDP Ping

## Task 3: Understanding Host Discovery through TCP/IP Layer


<img width="1024" height="800" alt="image" src="https://github.com/user-attachments/assets/4081f82a-c95b-496e-b445-53f4760958c6" />

### TCP/IP Layers Used by Nmap
<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/61005583-22d4-42af-a110-8c7d64d2b443" />


Nmap mainly uses:
- ARP
- ICMP
- TCP
- UDP


### 1. ARP (Link Layer)
**ARP (Address Resolution Protocol)** is used only on the local network (same subnet).

Purpose:
- Find a device's MAC address
- Confirm that a host is online

Example:
<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/303983e7-d232-420c-8b23-227318dba177" />


Advantages
- Very fast
- Very reliable
- Cannot cross routers


### 2. ICMP (Network Layer)
**ICMP (Internet Control Message Protocol)** is used for ping.

Nmap commonly uses:
- ICMP Echo Request (Type 8)
- ICMP Echo Reply (Type 0)

Example:
<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/8548dc47-f2d8-4e37-af57-98923223300b" />

If the target replies:
````
Host is Up
````


If no reply:
````
Host may be down
````
or
````
Firewall blocked ICMP
````




### 3. TCP (Transport Layer)
Sometimes firewalls block ICMP.

Instead, Nmap sends **TCP packets** to common ports.

Common ports:
- 22 (SSH)
- 80 (HTTP)
- 443 (HTTPS)

Example:

<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/4400365c-19f5-4582-ae2d-ab0528043e01" />


### 4. 4. UDP (Transport Layer)
Nmap can also use UDP.

Common ports:
- 53 (DNS)
- 67 (DHCP)
- 123 (NTP)


Example:

<img width="700" height="800" alt="image" src="https://github.com/user-attachments/assets/2545c7d0-3a51-478b-90fe-d5116d12e1c8" />

Even an ICMP Port Unreachable message indicates that the host itself is alive.


### Which Protocol Does Nmap Use?

| **Situation**    | **Protocol** |
| ---------------- | ------------ |
| Same subnet      | ARP          |
| Different subnet | ICMP         |
| ICMP blocked     | TCP          |
| TCP blocked      | UDP          |


### TCP/IP vs OSI Model
| OSI Layer   | TCP/IP Layer | Example Protocols          |
| ----------- | ------------ | -------------------------- |
| Application | Application  | HTTP, HTTPS, FTP, SSH, DNS |
| Transport   | Transport    | TCP, UDP                   |
| Network     | Internet     | IP, ICMP                   |
| Data Link   | Link         | ARP, Ethernet              |
| Physical    | Physical     | Cable, Wi-Fi               |

## Task 5: Nmap Host Discovery Using ARP

### What is ARP?
**ARP (Address Resolution Protocol)** is used to find the MAC address of a device from its IP address.
<img width="1024" height="700" alt="image" src="https://github.com/user-attachments/assets/210f8fc0-11c3-421f-a440-8aec971ed5d7" />


### When Does Nmap Use ARP?
Nmap automatically uses ARP when:
- The target is on the same subnet (LAN).
- We're running the scan with the necessary privileges (e.g., ``sudo`` on Linux).


### ARP Scan Command
````
sudo nmap -PR -sn 10.200.6.0/24
````
<img width="830" height="227" alt="image" src="https://github.com/user-attachments/assets/1c69cf81-a5cc-4cbe-ba11-2060bdf7f70e" />


#### Options
| Option | Meaning                            |
| ------ | ---------------------------------- |
| `-PR`  | ARP Ping Scan                      |
| `-sn`  | Host Discovery Only (No Port Scan) |

<img width="1024" height="596" alt="image" src="https://github.com/user-attachments/assets/610b53bd-32b2-4973-a77c-2e029befd136" />

#### How ARP Scan Works

##### Step 1
Nmap sends an ARP Request.
````
Who has 10.200.6.50?
````

##### Step 2
If the host is online, it replies.
````
I am 10.200.6.50
My MAC is 00:11:22:33:44:55
````

##### Step 3
Nmap marks the host as Up.
````
Host is up
````

#### Why ARP is Fast
ARP works at Layer 2 (Data Link Layer).

It:
- Doesn't need routing.
- Doesn't use TCP or UDP.
- Only checks devices on the local network.

This makes it **very fast and reliable**.

sudo nmap -PR -sn 10.48.132.228/24

<img width="643" height="117" alt="image" src="https://github.com/user-attachments/assets/2cdd9b7a-0427-4899-8f9a-13b768c82593" />


Why `1` is correct?

- When you scan a subnet like 10.48.132.238/24, Nmap checks 256 IPs (from 10.48.132.0 to 10.48.132.255).
- `Hosts up` means how many of those IPs actually replied.
In we case, only ``1`` machine answered, so Nmap said ``1`` host alive.
The rest didn’t respond — maybe they’re off, don’t exist, or are blocked by a firewall.

> Think of it like knocking on 256 doors: only one person opened the door. That’s why the answer is 1.

## Task 7: Nmap Host Discovery Using TCP and UDP
When **ICMP (ping)** is blocked by a firewall, Nmap can use TCP or UDP packets to determine whether a host is online.


### 1. TCP SYN Ping (-PS)

## Task 8: Using Reverse DNS Lookup (Important Points

Reverse DNS (rDNS) is the opposite of normal DNS.
- **Normal DNS**: Domain ➜ IP
- **Reverse DNS**: IP ➜ Domain (Hostname)


### Normal DNS
Find the IP address of a domain
````
example.com
      │
      ▼
93.184.216.34
````



### Reverse DNS (rDNS)
Find the hostname from an IP address:
````
10.200.6.15
      │
      ▼
mail.company.local
````
This helps identify what a system might be used for.


### Nmap Reverse DNS Options
| Option          | Meaning                                                               | Example                                    |
| --------------- | --------------------------------------------------------------------- | ------------------------------------------ |
| `-R`            | Always perform Reverse DNS lookup for every target (even if offline). | `nmap -R 10.200.6.0/24`                    |
| `-n`            | Disable DNS resolution (faster scan).                                 | `nmap -n 10.200.6.0/24`                    |
| `--dns-servers` | Use a specific DNS server.                                            | `nmap --dns-servers 8.8.8.8 10.200.6.0/24` |



### Commands

nmap -R 10.200.6.0/24

## Task 9: Summary

### Host Discovery Commands

| **Scan Type**              | **Purpose**                                                 | **Example Command**                         |
| -------------------------- | ----------------------------------------------------------- | ------------------------------------------- |
| **ARP Scan**               | Discover live hosts on the **same LAN** using ARP requests. | `sudo nmap -PR -sn 10.200.6.0/24`           |
| **ICMP Echo Scan**         | Send a normal **ping (Echo Request)**.                      | `sudo nmap -PE -sn 10.200.6.0/24`           |
| **ICMP Timestamp Scan**    | Send an **ICMP Timestamp Request**.                         | `sudo nmap -PP -sn 10.200.6.0/24`           |
| **ICMP Address Mask Scan** | Send an **ICMP Address Mask Request** (rarely used today).  | `sudo nmap -PM -sn 10.200.6.0/24`           |
| **TCP SYN Ping**           | Send a **TCP SYN** packet (default port 80).                | `sudo nmap -PS22,80,443 -sn 10.200.6.0/30`  |
| **TCP ACK Ping**           | Send a **TCP ACK** packet.                                  | `sudo nmap -PA22,80,443 -sn 10.200.6.0/30`  |
| **UDP Ping**               | Send a **UDP** packet to detect live hosts.                 | `sudo nmap -PU53,161,162 -sn 10.200.6.0/30` |


### Common Nmap Options
| **Option** | **Meaning**                            |
| ---------- | -------------------------------------- |
| **`-PR`**  | ARP Ping                               |
| **`-PE`**  | ICMP Echo (Normal Ping)                |
| **`-PP`**  | ICMP Timestamp Request                 |
| **`-PM`**  | ICMP Address Mask Request              |
| **`-PS`**  | TCP SYN Ping                           |
| **`-PA`**  | TCP ACK Ping                           |
| **`-PU`**  | UDP Ping                               |
| **`-sn`**  | Host discovery only (**no port scan**) |
| **`-n`**   | Disable DNS resolution (faster)        |
| **`-R`**   | Force Reverse DNS lookup for all hosts |


### When to Use Each Method
| **Situation**            | **Recommended Scan** |
| ------------------------ | -------------------- |
| Same local network (LAN) | `-PR` (ARP Scan)     |
| Different network        | `-PE` (ICMP Echo)    |
| Echo ping blocked        | `-PP` (Timestamp)    |
| Need TCP-based discovery | `-PS` (TCP SYN)      |
| Firewall filtering       | `-PA` (TCP ACK)      |
| Discover UDP-only hosts  | `-PU` (UDP Ping)     |
