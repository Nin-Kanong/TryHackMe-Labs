## Task 4 – General / Useful Utilities

This task introduces 4 useful Linux utilities you'll use frequently:
- ``wget`` → Download files
- ``scp`` → Copy files securely over SSH
- ``python3 -m http.server`` → Share files from your computer
- ``wget + Python HTTP Server`` → Download files from another machine


### 1. wget (Download Files)
``wget`` downloads a file from a website or web server.

#### Syntax:
````
wget <URL>
````

#### Example:
````
wget https://assets.tryhackme.com/additional/linux-fundamentals/part3/myfile.txt
````
Result:
````
Internet
    │
    │ HTTP Request
    ▼
Web Server
    │
    │ Sends myfile.txt
    ▼
Our Linux Machine
````
The downloaded file is saved in our current directory.

## Task 6 – Maintaining Your System: Automation (Cron Jobs)

### What is Cron?
Cron is a Linux service that automatically runs commands or scripts at scheduled times.

Examples:
- Backup files every day
- Update software every week
- Send reports every morning

### Edit Cron Jobs
Open our personal cron table:
````
crontab -e
````
<img width="1089" height="524" alt="image" src="https://github.com/user-attachments/assets/bb572264-0852-46ef-825a-694785b737ad" />


<img width="310" height="93" alt="image" src="https://github.com/user-attachments/assets/eca5da13-4f74-44ae-b73f-119d8d9014c8" />

This opens an editor (Nano/Vim) where you can add scheduled tasks.

#### Cron Format
````
* * * * * command
│ │ │ │ │
│ │ │ │ └── Day of Week (0-7)
│ │ │ └──── Month (1-12)
│ │ └────── Day of Month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
````

Helpful Websites

Generate cron expressions easily:
- Cron Guru
- Crontab Generator

#### Cron Fields
| Field | Meaning        | Example      |
| ----- | -------------- | ------------ |
| MIN   | Minute         | `30`         |
| HOUR  | Hour           | `14` (2 PM)  |
| DOM   | Day of Month   | `15`         |
| MON   | Month          | `7` (July)   |
| DOW   | Day of Week    | `1` (Monday) |
| CMD   | Command to run | `backup.sh`  |

#### Command:
| Command                  | Purpose                                     |
| ------------------------ | ------------------------------------------- |
| `crontab -e`             | Edit your cron jobs                         |
| `crontab -l`             | List (view) your cron jobs                  |
| `crontab -r`             | Remove all your cron jobs                   |
| `crontab -u username -e` | Edit another user's cron jobs (root only)   |
| `crontab -u username -l` | View another user's cron jobs (root only)   |
| `crontab -u username -r` | Remove another user's cron jobs (root only) |


````
Cron
 │
 ├── crontab -e      → Edit jobs
 ├── crontab -l      → List jobs
 ├── crontab -r      → Remove jobs
 │
 └── * * * * * cmd
      │ │ │ │ │
      │ │ │ │ └─ Day of Week
      │ │ │ └─── Month
      │ │ └───── Day of Month
      │ └─────── Hour
      └───────── Minute
````



### Wildcard (*)
The ``*`` means every.
| Cron         | Meaning                  |
| ------------ | ------------------------ |
| `* * * * *`  | Every minute             |
| `0 * * * *`  | Every hour               |
| `0 0 * * *`  | Every day at midnight    |
| `0 12 * * 1` | Every Monday at 12:00 PM |




### Backup Every 12 Hours
````
0 */12 * * * cp -R /home/cmnatic/Documents /var/backups/
````
Meaning:
| Part        | Meaning                          |
| ----------- | -------------------------------- |
| `0`         | At minute 0                      |
| `*/12`      | Every 12 hours                   |

## Task 7 – Maintaining Your System: Package Management

### What is APT?
**APT (Advanced Package Tool)** is Ubuntu's package manager.

It helps you:
- Install software
- Update software
- Remove software
- Manage repositories


### Package Repositories
A **repository (repo)** is an online storage location that contains software packages.
````
Ubuntu PC
     │
     │ apt
     ▼
Package Repository
     │
     ▼
Download Software
````
Ubuntu stores repository information in:
````
/etc/apt/
````
List the files:
````
ls /etc/apt
````
<img width="765" height="98" alt="image" src="https://github.com/user-attachments/assets/70c5f5bf-b1bc-4eb2-b8b7-6d82e79b45d9" />


#### Repository Files
| File/Folder                | Purpose                 |
| -------------------------- | ----------------------- |
| `/etc/apt/sources.list`    | Main repository list    |
| `/etc/apt/sources.list.d/` | Additional repositories |
| `trusted.gpg.d/`           | Trusted GPG keys        |



#### Add a Repository
Automatically:
````
sudo add-apt-repository ppa:<name>
````
Example:
````
sudo add-apt-repository ppa:graphics-drivers/ppa
````

Or we can manually:

Create a repo file:
````
cd /etc/apt/sources.list.d
sudo touch sublime-text.list
````

## Task 8 – Maintaining Your System: Logs

### What are Logs?
Logs are text files that record everything happening on your Linux system.

They help we:
- Troubleshoot problems
- Detect attacks
- Monitor services
- Debug applications

### Log Location
Most logs are stored in:
````
/var/log
````
List all logs:
````
cd /var/log
ls
````
<img width="1804" height="173" alt="image" src="https://github.com/user-attachments/assets/ec5d1f92-7792-4bdb-bda3-051de2dada8f" />


#### Common Log Files
| Log File                | Purpose                      |
| ----------------------- | ---------------------------- |
| `/var/log/syslog`       | General system logs          |
| `/var/log/auth.log`     | Login and authentication     |
| `/var/log/kern.log`     | Kernel messages              |
| `/var/log/dpkg.log`     | Package installation/removal |
| `/var/log/ufw.log`      | Firewall (UFW) logs          |
| `/var/log/fail2ban.log` | Failed login & banned IPs    |
| `/var/log/apache2/`     | Apache web server logs       |




### Apache Logs

### Linux Log Files & Useful Commands
| **Log File**           | **Command**                       | **Purpose**           | **Contains**                                              |
| ---------------------- | --------------------------------- | --------------------- | --------------------------------------------------------- |
| **System Log**         | `cat /var/log/syslog`             | General system events | System messages, services, kernel events                  |
| **Authentication Log** | `cat /var/log/auth.log`           | Authentication events | User logins, SSH logins, `sudo` usage, failed passwords   |
| **Kernel Log**         | `cat /var/log/kern.log`           | Kernel messages       | Hardware, drivers, kernel events                          |
| **Package Log**        | `cat /var/log/dpkg.log`           | Package management    | Software installation, upgrades, removals                 |
| **Firewall Log**       | `cat /var/log/ufw.log`            | UFW firewall activity | Allowed connections, blocked connections, firewall events |
| **Fail2Ban Log**       | `cat /var/log/fail2ban.log`       | Intrusion prevention  | Blocked IPs after repeated failed login attempts          |
| **Apache Access Log**  | `cat /var/log/apache2/access.log` | Web requests          | Successful HTTP requests                                  |
| **Apache Error Log**   | `cat /var/log/apache2/error.log`  | Web server errors     | Apache errors, missing files, permission issues           |


### Useful Log Commands
| **Command**            | **Purpose**                     | **Example**                    |
| ---------------------- | ------------------------------- | ------------------------------ |
| `cat filename`         | Display the entire file         | `cat /var/log/syslog`          |
| `head filename`        | Show the first 10 lines         | `head /var/log/auth.log`       |
| `tail filename`        | Show the last 10 lines          | `tail /var/log/syslog`         |
| `tail -f filename`     | Monitor a log file in real time | `tail -f /var/log/auth.log`    |
| `grep "text" filename` | Search for specific text        | `grep "error" /var/log/syslog` |


### Most Useful Commands
| **Command**                       | **Purpose**                      |
| --------------------------------- | -------------------------------- |
| `cd /var/log`                     | Go to the log directory          |
| `ls`                              | List all available log files     |
| `cat /var/log/syslog`             | View the system log              |
| `cat /var/log/auth.log`           | View authentication and SSH logs |
| `cat /var/log/apache2/access.log` | View Apache access log           |
| `cat /var/log/apache2/error.log`  | View Apache error log            |
| `cat /var/log/ufw.log`            | View firewall log                |
| `tail -f /var/log/syslog`         | Monitor the system log live      |
| `grep "error" /var/log/syslog`    | Search for error messages        |


### Log Directory Overview
| **Directory/File**   | **Purpose**                                   |
| -------------------- | --------------------------------------------- |
| `/var/log`           | Main directory for Linux log files            |
| `syslog`             | General system events                         |
| `auth.log`           | User authentication, SSH, and `sudo` activity |
| `kern.log`           | Kernel and hardware messages                  |
| `dpkg.log`           | Package installation and removal history      |
| `ufw.log`            | Firewall activity (UFW)                       |
| `fail2ban.log`       | Blocked IP addresses and login protection     |
| `apache2/access.log` | Successful web requests                       |
| `apache2/error.log`  | Apache web server errors                      |
