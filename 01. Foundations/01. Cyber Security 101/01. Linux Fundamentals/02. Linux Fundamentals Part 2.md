## Task 3: Introduction to Flags and Switches

### What are Flags/Switches?
**Flags (or switches)**: are extra options you add to a command to change what it does.

#### Syntax:
````
command -option
````
or
````
command --option
````

#### Example: ls
| Command     | What it does                                    |
| ----------- | ----------------------------------------------- |
| `ls`        | Show files and folders in the current directory |
| `ls -a`     | Show **all** files, including hidden files      |
| `ls --help` | Show available options and usage                |
| `man ls`    | Open the full manual for the `ls` command       |

## Task 5: Permissions 101
Linux uses ``permissions`` to control who can ``read``, ``write``, or ``execute`` a file or folder.


### 1. View Permissions
Use ``ls -lh`` (or ``ls -l``) to see permissions.

````
ls -lh
````
<img width="690" height="192" alt="image" src="https://github.com/user-attachments/assets/f7e6b51e-c430-4444-8880-9bbff174c66d" />


### 2. Permission Letters
| Letter | Meaning | What it allows       |
| ------ | ------- | -------------------- |
| `r`    | Read    | View the file        |
| `w`    | Write   | Edit the file        |
| `x`    | Execute | Run the file/program |


### 3. Permission Format
````
rwxr-xr-x
````
Split into three groups:
| Group   | Applies To | Example |
| ------- | ---------- | ------- |
| First 3 | Owner      | `rwx`   |
| Next 3  | Group      | `r-x`   |
| Last 3  | Others     | `r-x`   |


### 4. Switch User
Change to another user account:
````
su user2
````
<img width="922" height="102" alt="image" src="https://github.com/user-attachments/assets/752d5aa0-241f-4335-8a5f-33c4ccac6ce4" />

<img width="923" height="119" alt="image" src="https://github.com/user-attachments/assets/769e07c3-d253-4f8b-931a-04b2982f6ba0" />

- We need the ``user's password``.



### Numeric Permissions (chmod)
Each permission has a number:
| Permission    | Number |
| ------------- | ------ |
| Read (`r`)    | **4**  |
| Write (`w`)   | **2**  |
| Execute (`x`) | **1**  |

Add the numbers together for each group.

### Common Permission Values
| Symbolic    | Numeric | Meaning                                            |
| ----------- | ------- | -------------------------------------------------- |
| `rwxrwxrwx` | **777** | Everyone has full access                           |
| `rwxr-xr-x` | **755** | Owner full, others read & execute                  |
| `rw-r--r--` | **644** | Owner read/write, others read only                 |
| `rwx------` | **700** | Only owner has access                              |
| `rwxr-x---` | **750** | Owner full, group read & execute, others no access |


### Example Calculations
| Permission | Calculation | Number |
| ---------- | ----------- | ------ |
| `rwx`      | 4 + 2 + 1   | **7**  |
| `rw-`      | 4 + 2       | **6**  |
| `r-x`      | 4 + 1       | **5**  |
| `r--`      | 4           | **4**  |
| `---`      | 0           | **0**  |


### Change Permissions
Use ``chmod`` to change permissions.

````
chmod 755 file
````
Verify:
````
ls -ld file
````
<img width="673" height="130" alt="image" src="https://github.com/user-attachments/assets/2781f509-ca1c-4876-a700-f39d410a950f" />

| Part  | Meaning                            |
| ----- | ---------------------------------- |
| `d`   | Directory                          |
| `rwx` | Owner has Read, Write, Execute (7) |
| `r-x` | Group has Read & Execute (5)       |
| `r-x` | Others have Read & Execute (5)     |

Meaning:
| User   | Permission           |
| ------ | -------------------- |
| Owner  | Read, Write, Execute |
| Group  | Read, Execute        |
| Others | Read, Execute        |


````
chmod 750 system_overview.txt
````
<img width="753" height="183" alt="image" src="https://github.com/user-attachments/assets/b9006717-16bb-472a-aa11-1c20f7b025ff" />

No output means the command ``succeeded``.

#### Verify the permissions
````
ls -l system_overview.txt
````
<img width="760" height="74" alt="image" src="https://github.com/user-attachments/assets/451f4578-9e83-4a1b-965d-f16f63ebb10e" />

Meaning:
- **Owner**: Full access (rwx)
- **Group**: Read + Execute (r-x)
- **Others**: No access (---)


### Options:
| Command          | Purpose                                |
| ---------------- | -------------------------------------- |
| `ls -l`          | Show file permissions                  |
| `su user`        | Switch user                            |
| `su - user`      | Switch user and load their environment |
| `chmod 755 file` | Change file permissions                |
| `chmod 644 file` | Owner can edit; others can only read   |
| `chmod 700 file` | Only owner has access                  |

## Tas 6: Common Linux Directories
These are some of the most important folders we'll see in Linux.
| Directory | Purpose                                             | Example Contents              |
| --------- | --------------------------------------------------- | ----------------------------- |
| `/etc`    | Stores **system configuration files**               | `passwd`, `shadow`, `sudoers` |
| `/var`    | Stores **changing data** (logs, databases, cache)   | `log`, `tmp`, `backups`       |
| `/root`   | Home directory of the **root (administrator)** user | Root's personal files         |
| `/tmp`    | Stores **temporary files**                          | Temporary scripts, downloads  |
