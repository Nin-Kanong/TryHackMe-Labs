### 2. **grep** – Searching inside files
First in this I test to create one file to test:

To save it press `Ctrl + O` -> `Enter ` -> `Ctrl + X`.


grep looks for text patterns inside files.
````
grep "172.20.155.246" grep.txt
````
<img width="567" height="163" alt="image" src="https://github.com/user-attachments/assets/0bb6f356-0ef8-44f8-b92f-10c77f6aa17e" />

This shows only the lines in access.log that contain that IP address.

Or we can flter text:
````
grep "Hello" grep.txt
````
<img width="648" height="109" alt="image" src="https://github.com/user-attachments/assets/b605e491-e604-4f7b-b538-76734e9b77b9" />


##### Why not just cat?
- ``cat access.log`` shows everything → too much to read.
- ``grep`` filters only what you care about.

##### Searching recursively with -R:

If we don’t know which file contains the text, search all files in a folder and subfolders:
````
grep -R "PRETTY_NAME" /etc/
````
<img width="781" height="527" alt="image" src="https://github.com/user-attachments/assets/fbe29a58-d67c-4555-b2ec-8ad4bb5b5fae" />

- Shows the **file path** + the **matching line**.

##### Common useful options:
| Option       | Meaning                                      | Example                    |
| ------------ | -------------------------------------------- | -------------------------- |
| `-i`         | Ignore case (find `error`, `Error`, `ERROR`) | `grep -i "error" log.txt`  |
| `-n`         | Show line numbers                            | `grep -n "error" log.txt`  |
| `-r` or `-R` | Search recursively through directories       | `grep -R "error" /var/log` |
| `-l`         | Only show filenames with matches             | `grep -l "error" *.txt`    |


Example:
````
grep -ril "password" ~/Documents/
````
<img width="476" height="165" alt="image" src="https://github.com/user-attachments/assets/8c0207c8-db4b-4606-b256-e8e290978d59" />

Search case-insensitively, recursively, showing only filenames that contain ``password``.

##### Quick comparison table
| Command | What it searches       | Example                 |
| ------- | ---------------------- | ----------------------- |
| `find`  | Names of files/folders | `find -name "*.jpg"`    |
| `grep`  | Contents inside files  | `grep "TODO" notes.txt` |


##### Memory trick
- ``find`` = find file (name).
- ``grep`` = Global Regular Expression Print – search inside text.

> If you’re looking for a file → use find
> If you’re looking for text inside a file → use grep

## Task 7: An Introduction to Shell Operators 

### 1. **&** – Run a command in the background
Normally, when you run a command, your terminal is blocked until it finishes.
With ``&``, the command runs behind the scenes, and you can keep using the terminal.

````
sleep 30 &
````
<img width="538" height="245" alt="image" src="https://github.com/user-attachments/assets/4e73eca0-a24b-4e96-b9a3-e0b5c652ff2a" />

- ``sleep 30`` waits 30 seconds.
- The ``&`` runs it in the background.
- You immediately get the prompt back to run other commands.

##### When userful:
- Copying a large file
- Stating a server
- Long-running scripts



### 2. **&&** – Run multiple commands in a row, only if previous succeeded

- ``command1 && command2`` → ``command2`` runs only if ``command1`` finished successfully (exit code 0).

````
mkdir Test && cd Test
````
<img width="376" height="152" alt="image" src="https://github.com/user-attachments/assets/70f93d67-1515-4405-b7bc-3b095810c047" />

- If ``mkdir test`` works → then ``cd test`` runs.
- If ``mkdir test`` fails (e.g., permission denied) → ``cd test`` never runs.

````
nano makefile

````

<img width="817" height="159" alt="image" src="https://github.com/user-attachments/assets/662aa752-8dd3-4adf-83f8-28fe81d6f5b7" />

Save: ``CTRL + O`` -> ``Enter`` -> ``CTRL + X``.

<img width="466" height="307" alt="image" src="https://github.com/user-attachments/assets/a00f3e3a-2dfd-4592-853b-9a1e42b4cb2e" />

### 3. **>** – Redirect output to a file (overwrites)
Instead of printing to the terminal, ``>`` sends output to a file. Warning: If the file exists, it gets overwritten (old content lost).
````
echo hey > welcome
cat welcome
````
<img width="385" height="254" alt="image" src="https://github.com/user-attachments/assets/e98e5927-1772-4a98-ab18-aa1ade37e97f" />

- Creates ``welcome`` file containing ``hey``.
- If ``welcome`` already had text, it’s gone – replaced by ``hey``.

````
ls > files.txt
````
<img width="374" height="237" alt="image" src="https://github.com/user-attachments/assets/5e0f94f1-a8ef-4407-aca8-bb4e466df01f" />

- Saves directory listing to ``files.txt`` instead of showing on screen.




### 4. **>>** – Redirect and append to a file (no overwrite)
Same as ``>``, but adds to the end of the file instead of replacing.

- Start with
````
echo Hello > welcome   # welcome contains: hey
````
<img width="423" height="480" alt="image" src="https://github.com/user-attachments/assets/21028099-b7ac-4f66-b5b6-29be26fd3116" />

In this if we use only one `>` the new text that we input will replace our old text.

If we add `>>`, Now append:
````
echo hello >> welcome
cat welcome
````
<img width="375" height="174" alt="image" src="https://github.com/user-attachments/assets/4fcb9086-f203-4f43-9f99-c2a5e3e35a94" />

Now `welscome` conain:
````
hey
hello
````


##### Use for 
- Log files
- Collecting results from multiple commands.

##### Quick Comparison Table
| Operator | Effect                                     | Overwrites?                     | Runs in background? |
| -------- | ------------------------------------------ | ------------------------------- | ------------------- |
| `&`      | Background execution                       | N/A                             | ✅ Yes               |
| `&&`     | Run next command only if previous succeeds | N/A                             | ❌ No                |
| `>`      | Redirect output to file                    | ✅ Yes (replaces file content)   | ❌ No                |
| `>>`     | Redirect and append output to file         | ❌ No (adds to existing content) | ❌ No                |
