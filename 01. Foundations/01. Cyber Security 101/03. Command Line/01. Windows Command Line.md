### GUI vs CLI

Windows can be controlled in two ways.

| **GUI (Graphical User Interface)** | **CLI (Command Line Interface)**       |
| ---------------------------------- | -------------------------------------- |
| Click with a mouse                 | Type commands                          |
| Easy for beginners                 | Faster for experienced users           |
| Uses windows, icons, and menus     | Uses text commands only                |
| Good for everyday tasks            | Good for administration and automation |


### Why use the CLI?

The room gives three main reasons:

| **Reason**               | **Explanation**                                                                                            |
| ------------------------ | ---------------------------------------------------------------------------------------------------------- |
| **Lower resource usage** | CLI uses less CPU and RAM than GUI, so it works well on older or low-powered systems.                      |
| **Automation**           | You can create scripts (`.bat` or PowerShell scripts) to perform repetitive tasks automatically.           |
| **Remote management**    | You can connect to another computer (using SSH, for example) and manage it without being physically there. |

#### What does | mean?

The ``|`` symbol is called a pipe.

It sends the output of one command to another command.
````
systeminfo
      │
      ▼
     more
````
So:
````
systeminfo | more
````
Show the output of systeminfo one page at a time.

Press:
- ``Space`` → Next page
- ``Enter`` → Next line
- ``Ctrl + C`` → Stop viewing

#### 4. tracert: Shows every router (hop) our packets pass through to reach the destination.

Syntax:
````

````

In this I test on my terminal:
````

````
<img width="860" height="432" alt="image" src="https://github.com/user-attachments/assets/d92098c3-9500-480d-a999-4a8fd2bae0b5" />


|  **Hop** | **IP/Host**                             | **Time** | **Meaning**                                                                                  |
| -------: | --------------------------------------- | -------: | -------------------------------------------------------------------------------------------- |
|    **1** | `192.168.100.1`                         |     1 ms | Your local router (Default Gateway).                                                         |
|    **2** | `100.70.149.254`                        |     3 ms | Your Internet Service Provider (ISP).                                                        |
|    **3** | `172.16.250.113`                        |     6 ms | Another internal ISP router.                                                                 |
|    **4** | `16509.sgw.equinix.com (27.111.228.87)` | 24–25 ms | Internet backbone/exchange router (Equinix).                                                 |
| **5–10** | `* * * Request timed out.`              |      N/A | Routers did not respond to `tracert` (this is common and doesn't always indicate a problem). |
|   **11** | `64.239.109.1`                          | 23–24 ms | Destination server (`tryhackme.com`) reached successfully.                                   |

##### Symbols
| **Symbol** | **Meaning**                                                                 |
| ---------- | --------------------------------------------------------------------------- |
| `*`        | No response from that router before the timeout.                            |
| `ms`       | Milliseconds (how long the packet took to reach that hop).                  |
| `Hop`      | A router or device the packet passes through on its way to the destination. |



##### What is a Hop?
Every router your packet passes through is called a hop.
````
Our PC
   ↓
Router
   ↓
ISP
   ↓
TryHackMe
````
That's 3 hops.

## Task 5: Task and Process Management:

### Task and Process Management
A process is simply a program that is currently running.

Example:
| Program        | Process        |
| -------------- | -------------- |
| Google Chrome  | `chrome.exe`   |
| Notepad        | `notepad.exe`  |
| Command Prompt | `cmd.exe`      |
| File Explorer  | `explorer.exe` |


#### 1. tasklist: Lists all running processes.
````
tasklist
````
<img width="800" height="1012" alt="image" src="https://github.com/user-attachments/assets/1616df8f-aec7-4616-b04f-4261533cb93b" />

| **Column**           | **Description**                                                                                                              |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **Image Name**       | The name of the process (program or system service) currently running.                                                       |
| **PID (Process ID)** | A unique number assigned by Windows to identify each running process.                                                        |
| **Session Name**     | Indicates where the process is running, such as **Console** (interactive user session) or **Services** (background service). |
| **Session#**         | The numeric ID of the session in which the process is running.                                                               |
| **Mem Usage**        | The amount of memory (RAM) currently being used by the process.                                                              |



#### 2. tasklist /? : Shows the help page for tasklist
````
tasklist /?
````
<img width="824" height="1022" alt="image" src="https://github.com/user-attachments/assets/a5d3d958-28a2-47ae-b45a-b507829a95bc" />


Displays all available options.



#### 3. Filter a Process
Instead of showing every process, you can search for one.

Syntax:
````
tasklist /FI "imagename eq <process.exe>"
````


Example:
````
tasklist /FI "imagename eq sshd.exe"
````
<img width="853" height="197" alt="image" src="https://github.com/user-attachments/assets/7a8d7798-70db-4ec5-9fce-23d22c9fc2b9" />


| Part        | Meaning                   |
| ----------- | ------------------------- |
| `/FI`       | Apply a filter            |
| `imagename` | Filter by executable name |
| `eq`        | Equals                    |
| `sshd.exe`  | Process to search for     |




#### 4. Kill a Process
To stop (terminate) a running process, use ``taskkill``.

Syntax:
````
taskkill /PID <PID> /F
````

Example:
````
taskkill /PID 2660 /F
````
<img width="827" height="1021" alt="image" src="https://github.com/user-attachments/assets/20d8acff-9680-4408-b38e-72dde26f60aa" />

<img width="592" height="87" alt="image" src="https://github.com/user-attachments/assets/7b9b6053-1e22-4b1b-8de7-1302ea438ca1" />


Windows ends the process with PID 4567.



##### Kill by Process Name
Instead of using the PID, you can use the executable name.

````
taskkill /IM conhost.exe /F
````
<img width="728" height="187" alt="image" src="https://github.com/user-attachments/assets/a2055195-7953-4e1e-bc74-62456568cb04" />

This will close `conhost.exe`.

And it terminate process.

##### Force Termination
Sometimes a process won't close normally.

````
taskkill /F /PID 4567
````

| Option | Purpose                      |
| ------ | ---------------------------- |
| `/F`   | Force termination            |
| `/PID` | Kill by Process ID           |
| `/IM`  | Kill by Image (process) name |
