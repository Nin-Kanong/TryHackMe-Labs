## Task 4: The File System

### What is a File System?

A file system controls how Windows:
- Stores files
- Organizes folders
- Finds data on a disk

Think:
````
Hard drive = Library
File System = Librarian
````
The file system keeps everything organized.


### Modern Windows uses NTFS

**NTFS** = **New Technology File System**

This is the default file system used in modern Windows.



### Older file systems

Before NTFS there were:
- FAT16 / FAT32
- HPFS

FAT still exists today in:
- USB drives
- SD cards
- MicroSD cards

But Windows computers usually use NTFS.



### NTFS is a journaling file system

NTFS keeps a log (journal) of changes.

If Windows crashes:
````
Power off suddenly
````
NTFS can use its logs to help recover files.

FAT cannot do this.



### NTFS advantages

NTFS supports:

- Files larger than 4GB
- File/folder permissions
- Compression
- Encryption (EFS)

### NTFS Permissions Explained

NTFS permissions are security settings in Windows that control who can access files and folders and what they can do with them.

These permissions are mainly used on NTFS file systems.

#### Main NTFS Permissions Table
| Permission               | What It Allows on Folders                      | What It Allows on Files         |
| ------------------------ | ---------------------------------------------- | ------------------------------- |
| **Read**                 | View files and subfolders                      | Open and view file contents     |
| **Write**                | Add new files and folders                      | Edit or save data to a file     |
| **Read & Execute**       | View and open folders/files and run programs   | Open and execute programs/files |
| **List Folder Contents** | See files and subfolders inside a folder       | Not used for files              |
| **Modify**               | Read, write, and delete files/subfolders       | Read, edit, and delete files    |
| **Full Control**         | Complete access including changing permissions | Complete access to the file     |


#### Simple Explanation of Each Permission

1. Read

Allows a user to:
- Open files
- View content
- See folder contents

Example: A student can open a PDF but cannot edit it.

2. Write

Allows a user to:
- Create files
- Save changes
- Add folders

Example: A user can create a Word document in a folder.

3. Read & Execute

Allows a user to:
- Read files
- Run applications/programs

Example: We can open and run chrome.exe.

4. List Folder Contents

Allows a user to:
- View what is inside a folder

Example: We can see filenames inside Documents but may not open all files.

5. Modify

Allows a user to:
- Read files
- Edit files
- Delete files

Example: An employee can update and remove documents.

6. Full Control

Highest permission level.

Allows a user to:
- Read
- Write
- Modify
- Delete
- Change permissions

Example: Administrator access.

#### Permission Levels (Lowest → Highest)
| Level | Permission           |
| ----- | -------------------- |
| 1     | Read                 |
| 2     | Write                |
| 3     | Read & Execute       |
| 4     | List Folder Contents |
| 5     | Modify               |
| 6     | Full Control         |


#### How to Check NTFS Permissions in Windows
| Step | Action                      |
| ---- | --------------------------- |
| 1    | Right-click the file/folder |
| 2    | Click **Properties**        |
| 3    | Open the **Security** tab   |
| 4    | Select a user or group      |
| 5    | View permissions            |

### Alternate Data Streams (ADS) 

#### What is AD?
Alternate Data Streams (ADS) is a feature of the Windows NTFS file system.

Normally, a file has one main data stream called:
````
$DATA
````
ADS allows a file to contain additional hidden streams of data without changing the main file content.

#### Simple Idea
Think of a file like a backpack:
| Part      | Meaning                                 |
| --------- | --------------------------------------- |
| Main file | Normal visible content                  |
| ADS       | Hidden extra pocket inside the backpack |


Exampe:
````
note.txt
````
This file can secretly contain another hidden stream:
````
note.txt:hidden.txt
````
The hidden stream is not normally visible in Windows Explorer.


#### ADS Table
| Feature             | Explanation                                |
| ------------------- | ------------------------------------------ |
| Full Name           | Alternate Data Streams                     |
| File System         | NTFS only                                  |
| Purpose             | Store extra hidden data inside files       |
| Visible in Explorer | Usually No                                 |
| Used By             | Windows, browsers, malware, security tools |
| Main Stream         | `$DATA`                                    |
