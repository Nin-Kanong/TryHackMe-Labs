## Task 4: The File System

### What is a File System?

A file system controls how Windows:
- Stores files
- Organizes folders
- Finds data on a disk

Think:
````
Hard drive = Library
File System = Librarian
````
The file system keeps everything organized.


### Modern Windows uses NTFS

**NTFS** = **New Technology File System**

This is the default file system used in modern Windows.



### Older file systems

Before NTFS there were:
- FAT16 / FAT32
- HPFS

FAT still exists today in:
- USB drives
- SD cards
- MicroSD cards

But Windows computers usually use NTFS.



### NTFS is a journaling file system

NTFS keeps a log (journal) of changes.

If Windows crashes:
````
Power off suddenly
````
NTFS can use its logs to help recover files.

FAT cannot do this.



### NTFS advantages

NTFS supports:

- Files larger than 4GB
- File/folder permissions
- Compression
- Encryption (EFS)

### NTFS Permissions Explained

NTFS permissions are security settings in Windows that control who can access files and folders and what they can do with them.

These permissions are mainly used on NTFS file systems.

#### Main NTFS Permissions Table
| Permission               | What It Allows on Folders                      | What It Allows on Files         |
| ------------------------ | ---------------------------------------------- | ------------------------------- |
| **Read**                 | View files and subfolders                      | Open and view file contents     |
| **Write**                | Add new files and folders                      | Edit or save data to a file     |
| **Read & Execute**       | View and open folders/files and run programs   | Open and execute programs/files |
| **List Folder Contents** | See files and subfolders inside a folder       | Not used for files              |
| **Modify**               | Read, write, and delete files/subfolders       | Read, edit, and delete files    |
| **Full Control**         | Complete access including changing permissions | Complete access to the file     |


#### Simple Explanation of Each Permission

1. Read

Allows a user to:
- Open files
- View content
- See folder contents

Example: A student can open a PDF but cannot edit it.

2. Write

Allows a user to:
- Create files
- Save changes
- Add folders

Example: A user can create a Word document in a folder.

3. Read & Execute

Allows a user to:
- Read files
- Run applications/programs

Example: We can open and run chrome.exe.

4. List Folder Contents

Allows a user to:
- View what is inside a folder

Example: We can see filenames inside Documents but may not open all files.

5. Modify

Allows a user to:
- Read files
- Edit files
- Delete files

Example: An employee can update and remove documents.

6. Full Control

Highest permission level.

Allows a user to:
- Read
- Write
- Modify
- Delete
- Change permissions

Example: Administrator access.

#### Permission Levels (Lowest → Highest)
| Level | Permission           |
| ----- | -------------------- |
| 1     | Read                 |
| 2     | Write                |
| 3     | Read & Execute       |
| 4     | List Folder Contents |
| 5     | Modify               |
| 6     | Full Control         |

### Alternate Data Streams (ADS) 

#### What is AD?
Alternate Data Streams (ADS) is a feature of the Windows NTFS file system.

Normally, a file has one main data stream called:
````
$DATA
````
ADS allows a file to contain additional hidden streams of data without changing the main file content.

#### Simple Idea
Think of a file like a backpack:
| Part      | Meaning                                 |
| --------- | --------------------------------------- |
| Main file | Normal visible content                  |
| ADS       | Hidden extra pocket inside the backpack |


Exampe:
````
note.txt
````
This file can secretly contain another hidden stream:
````
note.txt:hidden.txt
````
The hidden stream is not normally visible in Windows Explorer.


#### ADS Table
| Feature             | Explanation                                |
| ------------------- | ------------------------------------------ |
| Full Name           | Alternate Data Streams                     |
| File System         | NTFS only                                  |
| Purpose             | Store extra hidden data inside files       |
| Visible in Explorer | Usually No                                 |
| Used By             | Windows, browsers, malware, security tools |
| Main Stream         | `$DATA`                                    |

## The Windows\System32 Folders

### Windows folder 

* **`C:\Windows`** is the folder that usually contains the **Windows operating system files**.
* It does **not always have to be on the C: drive**; Windows can be installed elsewhere.
* Windows uses **environment variables** to locate important folders.

Example:

```
%windir%
```

`%windir%` = points to the Windows folder location.

Environment variables store system information such as:

* OS path
* Processor information
* Temporary folder locations

The **Windows folder** contains many important subfolders used by the operating system.

### **System32 Summary:**
* **System32** contains **critical Windows system files**.
* Be **very careful** — deleting files here can break Windows and make it unusable.
* Many important Windows tools are stored inside **System32**.

#### Go to:
**File Explorer** -> **This PC** -> **Local Disk (C:)** -> **Windows** -> **System32**:
<img width="1089" height="950" alt="image" src="https://github.com/user-attachments/assets/5b9c4525-4f20-4c62-9994-028806ba988d" />

<img width="1087" height="947" alt="image" src="https://github.com/user-attachments/assets/4eff6ed3-af31-4249-99b7-b70cba5e5317" />


The folder you highlighted is System32 inside Windows.

`C:\Windows\System32` is one of the most important folders in Microsoft Windows.

It stores core system files that Windows needs to run.

* system folders
* drivers
* configuration files
* language files
* Windows tools

Here is what System32 is used for:

| Type                      | Purpose                  | Example                      |
| ------------------------- | ------------------------ | ---------------------------- |
| System DLL files          | Core Windows functions   | `kernel32.dll`               |
| Executable tools          | Windows utilities        | `cmd.exe`, `taskmgr.exe`     |
| Drivers                   | Hardware communication   | Printer, keyboard, network   |
| Configuration             | System settings          | Group policies, boot configs |
| Admin tools               | Management utilities     | PowerShell, Event Viewer     |
| Authentication components | Login/security processes | LSASS-related files          |

Common important files in System32:

| File                 | What it does          |
| -------------------- | --------------------- |
| `cmd.exe`            | Command Prompt        |
| `WindowsPowerShell\` | PowerShell            |
| `taskmgr.exe`        | Task Manager          |
| `mstsc.exe`          | Remote Desktop client |
| `regedit.exe`        | Registry Editor       |
| `drivers\`           | Device drivers        |
| `config\`            | Registry hive files   |

Why hackers care about System32:

* It contains critical Windows binaries
* Attackers may:

  * replace files,
  * abuse built-in tools (“Living off the Land”),
  * dump credentials,
  * create persistence,
  * hide malware.

But normal users should usually:

* NOT delete files there
* NOT modify random files
* because Windows can break.


In this:
* `drivers` → hardware drivers
* `DriverStore` → official driver packages
* `config` → registry/system configuration
* `GroupPolicyUsers` → Group Policy settings
* language folders like `en-US`, `fr-FR`

So System32 is basically:

> the “engine room” of Windows.

## Task 6: User Account, Profiles, and Permissions

This explanation is talking about the two main account types in Microsoft Windows.

### Windows User Account Types

| Account Type  | Permissions    | What They Can Do                                                          |
| ------------- | -------------- | ------------------------------------------------------------------------- |
| Administrator | Full control   | Install software, change system settings, manage users, access most files |
| Standard User | Limited access | Use apps, change personal files/settings only                             |

### 1. Administrator Account

An Administrator has high privileges on the computer.

They can:

* Install/uninstall programs
* Add or delete user accounts
* Change passwords
* Modify security settings
* Access system folders like:

  * `C:\Windows`
  * `System32`
* Manage services and drivers

Examples:

* IT administrator
* Our own PC admin account

---

## 2. Standard User Account

A Standard User has restricted permissions for safety.

They usually:

* Cannot install software system-wide
* Cannot modify important Windows files
* Cannot change security settings
* Cannot create/delete users

They mainly:

* Browse the web
* Use applications
* Access personal documents

---

# Why This Matters in Cybersecurity

Attackers often first compromise:

* a Standard User account

Then try:

* privilege escalation
* credential dumping
* abusing misconfigurations

Goal:

> Become Administrator or Domain Admin.

Because Administrator access gives much more control over the system.

# 6. What Are Groups?

Groups help Windows manage permissions more easily.

Instead of assigning permissions one-by-one:

* users are added to groups
* groups already have permissions

Example:

| Group                | Permissions         |
| -------------------- | ------------------- |
| Administrators       | Full system control |
| Users                | Basic access        |
| Remote Desktop Users | Can use RDP         |
| Backup Operators     | Backup permissions  |

---

# 7. Permission Inheritance

If a user joins a group:

```
User → Group → Permissions
```

The user automatically inherits that group's permissions.

Example:

* Add John to Administrators
* John becomes admin immediately

---

# 8. Users Can Belong to Multiple Groups

Example:

```
k4n0ng
 ├── Users
 ├── Remote Desktop Users
 └── Backup Operators
```

`k4n0ng` receives permissions from all three groups.

---

# 9. Why This Matters in Cybersecurity

Attackers often look for:

* admin accounts
* misconfigured groups
* users inside powerful groups

Common targets:

* Administrators
* Domain Admins
* Remote Desktop Users
* Backup Operators

Because these can help:

* privilege escalation
* lateral movement
* persistence

## Task 7: User Account Control

This lesson is explaining User Account Control (UAC) in Microsoft Windows.

### What is UAC?
UAC is a security feature that helps prevent unauthorized system changes.

Even if you are logged in as an Administrator:

* Windows does NOT automatically run everything with full admin privileges.

Instead:

* Windows asks for confirmation before important actions.

That popup is called:

> User Account Control (UAC)

---

### What UAC Protects Against

Without UAC:

* malware could silently install programs
* change system settings
* disable antivirus
* create admin users

With UAC:

* Windows pauses and asks:

  > “Do you want to allow this app to make changes?”

---

#### What the Shield Icon Means

In your screenshot, the installer icon has a small shield.

That means:

| Icon        | Meaning                            |
| ----------- | ---------------------------------- |
| Shield icon | Requires elevated/admin privileges |
| No shield   | Can run as normal user             |

Programs like:

* installers
* registry editors
* admin tools

usually trigger UAC.

---

#### Standard User vs Administrator with UAC

| Account Type  | What Happens              |
| ------------- | ------------------------- |
| Administrator | Gets Yes/No confirmation  |
| Standard User | Must enter admin password |

That is exactly what your screenshot shows:

* Standard user tries to install Wireshark
* UAC asks for Administrator credentials

---

#### Important Note from the Lesson

The built-in:

```
Administrator
```

account behaves differently.

By default:

* UAC restrictions are reduced/disabled for the built-in Administrator account.

That is why security professionals recommend:

* using a normal admin account
* not using the built-in Administrator daily

---

#### Why Cybersecurity Students Learn This

Attackers often try to:

* bypass UAC
* trick users into clicking Yes
* escalate privileges

Because:

* admin privileges = much more system control

Common attack goals:

* disable antivirus
* install persistence
* dump credentials
* move laterally
